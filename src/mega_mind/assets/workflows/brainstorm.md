# Brainstorm Workflow

## Trigger

Use when starting a new feature or making architectural decisions.

**Quick Start:** `/mega-mind route "I need to design..."` or `/brainstorm`

## Steps

### 1. Define the Problem

- What are we trying to solve?
- Who are the stakeholders?
- What are the constraints?

### 2. Generate Options

- Create at least 3 different approaches
- Consider existing solutions
- Think about future scalability

### 3. Evaluate Options

- Compare against requirements
- Assess risks and benefits
- Consider implementation effort

### 4. Make a Decision

- Select the best approach
- Document the rationale
- Identify next steps

## Next Steps After Brainstorming

After completing this workflow, typically continue with:

```
/writing-plans → Create detailed implementation plan
```

Or use:

```
/mega-mind execute write-plan
```

## Output

- Decision document with all options considered
- Recommended approach with rationale
- Implementation plan outline
